# 基金自选列表二层交互规格

## 1 目标
将“滚动列表”升级为“看盘视角切换”，让“我的钱涨跌”与“市场发生了什么”之间通过上下滑动自然过渡，并在视觉与触感上提供“状态确认”。

## 2 核心结构
- **底层：市场背景层**（始终可见）：默认折叠为窄条，呈现三大指数 + 总体涨跌热力；上拉后展开完整面板，展示热门板块、资金流向、涨跌榜和市场解读。
- **上层：自选列表层**（可推拉）：包含自选基金列表、分组筛选、排序控件；默认占据屏幕 70%~80%，上滑时透明度降低并轻微缩放，产生被推开的效果。

## 3 状态定义
1. **基金优先（默认）**
   - 自选列表层覆盖 70%~80% 屏幕。
   - 市场层仅保留顶部窄条（含三大指数、涨跌概览）。
   - 适用于日常盯盘。
2. **市场优先**
   - 自选列表层收缩至顶部或全屏淡化，交互焦点转向市场层。
   - 市场层展开完整内容（板块/资金、涨跌榜、市场解读、焦点 ETF/基金提示）。
   - 适用于行情理解、调仓决策。

## 4 交互规则
- **向上滑动**：仅在列表抵达顶部（`scrollY ≤ 0`）并向上超出至少 30px 后才进入状态切换判断；此时上层逐渐透明并缩放，底层逐步展开。
- **触发阈值**：向上拖动 **≥ 140px**（约 67pt）即视为进入市场优先；向下拖动使其回到基金优先。
- **速度补偿**：释放速度 ≥ 1200px/s（任意方向）立即触发目标状态，避免快速手势被阻尼取消。
- **吸附策略**：定义“留在基金区”0~80px、“留在市场区”≥120px；释放在区间内则按速度决定，否则自动吸附到最近状态。
- **阻尼与最大位移**：中间拖动阶段施加 0.35 的阻尼（多拖 1px 仅移动 0.35px），并将最大偏移限制在 220px。
- **松手反馈**：完成阈值判断后启动 280ms ease-out 吸附动画，并在动画开始瞬间触发一次轻冲击触觉。

## 5 视觉联动
- **自选层**：透明度线性从 1.0 → 0.6；缩放从 1.0 → 0.96；过渡在 80px 之后渐进。
- **市场层**：透明度从 0.5 → 1.0，模糊逐步消失；缩放从 0.96 → 1.0，与自选层动画同步。
- **顶部窄条**：作为过渡锚点，始终可见；用于串联两态。
- **触觉反馈**：吸附动画触发时提供确认感，增强“完成状态切换”的认知。

## 6 内容辅佐
- **收缩态市场层**：三大指数、当日涨跌热力仪表。
- **展开态市场层**：
  - 热门板块 TOP3
  - 资金流向 / 涨跌比
  - 1~2 条简短市场解读（AI 简述）
  - 选填：焦点 ETF/基金联动提示
- **自选列表**：信息密度提升，突出估算涨幅、净值、日涨幅。次要指标提供折叠或横向滑动以展开，保持盯盘友好。
- **分组筛选与排序控件**：固定在自选层顶部，不参与透明度/缩放变化，确保筛选行为稳定。

## 7 边界与容错
- **长列表滚动冲突**：只有在顶部停稳且上拉超过 30px 后才进入状态判断，避免误触。
- **空自选场景**：自动将市场层渲染为 60% 高度，默认市场优先；此时向上无效，向下可切换到约 40px 窄条，引导用户关注行情后加入自选。
- **分组筛选/排序**：始终留在自选层，可操作但不参与视觉淡化，避免状态变化中断操作。

## 8 可交付文档建议
1. **交互阈值表**：记录触发距离、速度门槛、阻尼、最大位移、吸附区间、动画时长/曲线、触觉时机。
2. **两态布局图稿**：组件尺寸、层级关系、透明度/缩放曲线、内容优先级、物料占比。
3. **iOS 实现方案**：基于 `ScrollView + offset + PreferenceKey` 的手势/状态管理说明，配合上述阈值与动画参数。
